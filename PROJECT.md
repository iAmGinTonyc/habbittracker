# Habit Tracker — выгрузка по проекту

Минималистичный трекер привычек в чёрно‑белом стиле, с лёгкой геймификацией
(опыт/уровни, влияющие только на разблокировку мини‑игр), утренними/вечерними
чек‑апами и месячной историей в виде тепловой карты.

Дата выгрузки: 22 июня 2026.

---

## 1. Технологии и структура файлов

- **Чистый фронтенд без зависимостей и сборки.** Vanilla JS, HTML, CSS. Графики
  рисуются вручную на `<canvas>` (без Chart.js и т.п.). Открывается как статика.
- **Хранение — `localStorage`**, один ключ `habbittracker_progress` (JSON).
  Бэкенда нет, всё локально в браузере.

```
habbittracker-main/
├── index.html          — разметка всех экранов и модалок
├── habbittracker.css   — вся стилизация (дизайн‑система ч/б)
├── habbittracker.js    — вся логика (≈1500 строк, один IIFE на DOMContentLoaded)
├── pics/               — изображения (волк‑мастер, игральные карты, и пр.)
├── PROJECT.md          — этот документ
└── .claude/launch.json — конфиг локального превью (python http.server)
```

Запуск локально: `python3 -m http.server 4599` в папке проекта, затем открыть
`http://localhost:4599`.

## 2. Поток приложения (экраны)

```
Интро (волк‑мастер + фраза)
        │  тап по экрану
        ▼
  Дашборд ──► нижний таб‑бар из 5 вкладок:
        ├── День      (привычки на сегодня)         ← стартовая
        ├── Месяц     (история / тепловая карта)
        ├── Игры      (мини‑игры, часть закрыта)
        ├── Утро      (утренний чек‑ап)
        └── Вечер     (вечерний чек‑ап)
```

- **Новый пользователь:** видит интро → тап → создаются 10 привычек по умолчанию →
  попадает на «День».
- **Вернувшийся пользователь:** интро пропускается, сразу «День».
- Экранов выбора идентичности/целей и «эволюции» **нет** (удалены).

## 3. Блоки и экраны — детально

### 3.1. Интро
- Изображение медитирующего волка‑мастера (`pics/wolf_master.jpeg`, grayscale),
  плавающая анимация.
- Мотивирующая фраза, меняется каждые 5 секунд (3 фразы по кругу).
- Подсказка «нажми на экран…». Тап → загрузочный оверлей → «День».

### 3.2. День (вкладка «День»)
Главный экран трекинга.
- **Шапка (общая для всех вкладок):** слева — текущая дата («22 июня,
  понедельник»), справа — «УРОВЕНЬ N». В левом верхнем углу — кнопка сброса `↺`.
- **Заголовок** «Задачи на сегодня» + таймер «до обновления: ЧЧ:ММ:СС»
  (отсчёт до полуночи, когда отметки сбрасываются).
- **Список привычек:** строка = чёрно‑белый чекбокс + название + (серия) + «⋯».
  - Клик по чекбоксу или названию — отметить/снять выполнение за сегодня.
  - Выполненная привычка: залитый чёрный чекбокс с галочкой + зачёркивание.
  - Если есть серия — справа моно‑огонёк + число.
  - «⋯» открывает модалку настроек привычки.
- **Добавление:** строка с полем «+ добавить привычку», ввод + Enter. Лимит — **10
  привычек**. При 10/10 поле заменяется надписью «Максимум 10 привычек».
- **Подвал (общий):** прогресс‑бар опыта до следующего уровня + «X / Y XP» и «%».

### 3.3. Месяц (вкладка «Месяц») — история / тепловая карта
- **Навигация по месяцам:** `← Июнь 2026 →`.
- **Сводка месяца:** «выполнено / возможно / прогресс %» + прогресс‑бар.
- **Тепловая карта (в стиле GitHub):** по строке на каждую привычку, по клетке на
  день месяца. Залитая клетка = выполнено, «сегодня» обведено рамкой, будущие дни
  заблокированы. Справа от строки — серия (моно‑огонёк) и счётчик `N/30`.
  - **Редактируемая:** клик по клетке отмечает/снимает день задним числом
    (правка влияет на лог и серии, но XP за это не начисляется).
- **График «Настроение и качество сна»:** линия настроения (сплошная) и качества
  сна (пунктир) за месяц — данные берутся из утренних чек‑апов.

### 3.4. Игры (вкладка «Игры»)
Три мини‑игры; открытость зависит от уровня (см. раздел 6).
- Закрытые игры показаны пунктиром с замком и подписью «Откроется с уровнем»
  + подсказка «Новые игры открываются на ур. 3 и 7».
- **Найди пару** — мемори: 8 пар (16 карт) из колоды игральных карт (`pics/`).
  Открой все пары. XP = `max(1, round(пары × 1.25))`, максимум ~10.
- **Посчитай** — устный счёт на время (60 c). Сложность: 1–9 / 10–99 / 100–999.
  XP = `max(1, min(10, верных))`.
- **10 слов** — запомнить набор слов за отведённое время, затем ввести.
  XP = `max(1, угадано)` (из 8).

### 3.5. Утро (вкладка «Утро») — утренний чек‑ап
Поля формы:
- Во сколько лёг в кровать? (время)
- Во сколько уснул? (время)
- Во сколько проснулся? (время)
- Качество сна (шкала 1–10)
- Уровень энергии утром (1–10)
- Настрой / Мотивация (1–10)
- Главный фокус на сегодня (текст)

Кнопки: «Сохранить чек‑ап», «Аналитика». После сохранения форма блокируется,
появляется «Редактировать». Иконка‑календарь в шапке — просмотр истории за другой
день. Первое сохранение за день даёт **+3 XP**.

### 3.6. Вечер (вкладка «Вечер») — вечерний чек‑ап
Поля формы:
- Общая оценка дня (1–10)
- Уровень энергии сейчас (1–10)
- Удовлетворённость результатами (1–10)
- Уровень спокойствия (1–10)
- Качество выполнения привычек (1–10)
- За что благодарен сегодня? (текст)
- Что улучшить завтра? (текст)

Поведение — как у утреннего (сохранение, блокировка, история, +3 XP).

### 3.7. Аналитика (модалка из чек‑апов)
По кнопке «Аналитика» открывается полноэкранная модалка: выбор диапазона дат и
линейные графики по каждому числовому показателю чек‑апа за период.

### 3.8. Модалки
- **Настройки привычки** (по «⋯»): редактирование названия, формула «Я сделаю это
  после того, как…» (триггер), переключатель «Напомнить в:» + время, и кнопка
  **«Удалить»** (с подтверждением; чистит и историю этой привычки).
- **Разблокировка игры** (на ур. 3 и 7): «Новая игра открыта — выбери, что
  добавить» со списком ещё закрытых игр.

### 3.9. Напоминания
Каждые 30 секунд проверяется: если у привычки задано время напоминания и оно
совпало с текущим, а привычка не выполнена — всплывает тост «Время действовать»
с названием привычки (и звук).

## 5. Серии (стрики)

- Серия привычки = число подряд идущих выполненных дней вплоть до сегодня.
- Если сегодня ещё не отмечено — серия **не рвётся**, отсчёт идёт со вчера.
- Считается из постоянного лога (`history`), показывается на «Дне» и в «Месяце».

## 6. Опыт, уровни и разблокировка игр

**Важно:** опыт и уровни сейчас влияют **только** на количество открытых мини‑игр.
На привычки, лимиты и прочее они не влияют.

### Что даёт XP
| Действие | XP |
|---|---|
| Выполнить привычку | `5 + (уровень − 1) × 3` (ур.1 → 5, ур.2 → 8, ур.3 → 11…) |
| Сохранить чек‑ап (первый раз за день) | +3 |
| Игра «Найди пару» | до ~10 |
| Игра «Посчитай» | до 10 |
| Игра «10 слов» | до 8 |

Защита от «фарма»: за одну привычку XP начисляется **не больше одного раза в день**
(повторное снятие/постановка галочки опыт не добавляет).

### Уровни
- Нужно XP до следующего уровня: `floor(15 × уровень^1.8)`
  (ур.1 → 15, ур.2 → 52, ур.3 → 108, далее растёт).
- При достижении порога: уровень +1, текущий XP обнуляется (без переноса остатка).

### Разблокировка игр
- На старте открыта только **«Найди пару»**.
- На **ур. 3** и **ур. 7** при повышении уровня всплывает выбор: какую из ещё
  закрытых игр открыть.
- На **ур. 10** новых игр пока нет (задел под будущую 4‑ю игру).

## 7. Модель данных (localStorage)

Ключ: `habbittracker_progress`. Один JSON‑объект:

```js
{
  level: 1,
  currentXP: 0,
  habits: [
    {
      uid: "u…",          // стабильный уникальный id привычки
      text: "Тренировка",
      completed: false,    // отметка за СЕГОДНЯ (сбрасывается на новый день)
      xpDate: "2026-06-22",// дата, когда за неё уже начислили XP (анти‑фарм)
      triggerText: "",     // «после того как…» (опц.)
      reminderTime: null   // "08:00" или null (опц.)
    }
  ],
  unlockedGames: ["memory"],            // открытые игры
  lastActiveDate: "2026-06-22",
  checkins: { morning: {}, evening: {} },          // черновик чек‑апов на сегодня
  checkinHistory: {                                // сохранённые чек‑апы по дням
    "2026-06-22": { morning: {…}, evening: {…} }
  },
  history: {                                       // постоянный лог выполнения
    "2026-06-22": { "u…": true }                   // дата → { uid: true }
  }
}
```

- `history` — это «вечная память» трекера: именно по нему строятся тепловая карта
  и серии. Он **не** очищается при смене дня.
- Смена дня (`checkNewDay`): на новую календарную дату обнуляются `completed` у
  привычек и черновик `checkins`; `history` и `checkinHistory` сохраняются.

## 8. Что можно делать на сайте (интеракции)

- Пройти интро тапом и получить стартовый набор привычек.
- Отмечать/снимать выполнение привычек за сегодня (чекбокс или название).
- Добавлять привычки (до 10), переименовывать, задавать триггер и время
  напоминания, удалять.
- Получать опыт и повышать уровень за привычки, чек‑апы и игры.
- На ур. 3 и 7 — выбирать, какую мини‑игру открыть.
- Играть в открытые мини‑игры (память, счёт, слова) и зарабатывать XP.
- Заполнять утренний и вечерний чек‑апы; редактировать их; смотреть за прошлые дни.
- Открывать аналитику чек‑апов (графики по диапазону дат).
- Смотреть месячную тепловую карту, листать месяцы, отмечать дни задним числом,
  видеть серии и график настроения/сна.
- Получать всплывающие напоминания по привычкам в заданное время.
- Сбросить весь прогресс кнопкой `↺` (с подтверждением).

## 9. Дизайн‑система

- **Палитра — чёрно‑белая.** Основные цвета: `#111` (чёрный), `#fff` (белый),
  оттенки серого (`#666`, `#999`, `#bbb`, `#e0e0e0`, `#f3f3f3`). Цветных акцентов
  нет (серии‑огоньки тоже монохромные).
- **Шрифт:** Inter / system‑ui.
- **Иконки:** тонкие монохромные SVG (таб‑бар, календарь, «троеточие», замок,
  огонёк). Эмодзи в интерфейсе убраны.
- **Адаптив:** одноколоночная мобильная вёрстка; таб‑бар из 5 вкладок
  (иконка + подпись), активная вкладка — чёрная.

## 10. Что можно доработать дальше (идеи и мелочи)

**Идеи развития**
- 4‑я мини‑игра, чтобы порог ур. 10 тоже что‑то открывал.
- Экспорт/импорт данных (JSON), синхронизация между устройствами (нужен бэкенд).
- Напоминания через настоящие Web Notifications (сейчас — только тост в открытой
  вкладке, и только пока сайт открыт).
- Гибкая периодичность привычек (не только «каждый день»).
- Перенос несгораемого остатка XP при повышении уровня (сейчас обнуляется).

**Мелочи/тех‑долг**
- Тост напоминания всё ещё содержит эмодзи 🔔 — можно заменить на моно‑иконку для
  полного соответствия ч/б стилю.
- В `pics/` остались неиспользуемые ассеты: волки `Sneaky wolf 1–6` (анимация
  уровня удалена), `Красный/Чёрный Джокер`, `Рубашка` (мемори берёт только 52
  обычные карты). Можно удалить для чистоты.
- Дата «сегодня» в логе привычек считается по локальному времени, а часть старой
  логики чек‑апов — по UTC; на стыке полуночи в редких часовых поясах возможно
  расхождение в один день (данные при этом не теряются).
